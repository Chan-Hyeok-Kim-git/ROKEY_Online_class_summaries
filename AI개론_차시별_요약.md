# 04장: 인공지능 기초 - 신경망과 역전파
지도학습 → AI에게 입력과 정답이 쌍으로 주어진 데이터로 학습(분류(스팸 메일), 회귀(주택 가격 예측, 매출 예측))<br>
비지도학습 → 데이터만 주고 정답없이 데이터의 숨겨진 패턴을 스스로 발견(군집화(소매패턴에 따른 고객 그룹화),차원축소(복잡한데이터 2D로 시각화))<br>
자기지도학습 → 책의 일부 가리고 다음 단어 맞추며 배우듯, 데이터 자체에서 자동으로 레이블 만들어 학습<br>
(Masked Learning(글씨 가리고 학습), Contrastive Learning(같은 고양이 사진(다른 각도)은 비슷함)); BERT, ViT, GPT<br>
강화학습 → '시행착오' 통해 보상 최대화 하는 행동 학습(상태 파악, 행동, 행동 이후 보상(특정 행동시 보상줘 제일 많은 보상을 얻는 경로로 행동))

### 선형모델 → 모든 딥러닝의 시작<br>
입력(X)과 가중치(W)의 선형 결합 → y = wx +b<br>
1. 문제 정의 → 입력(X)과 정답(Y)의 명확한 정의가 제일 우선, 예측 함수 목적 및 최종 출력 형태 결정(과업의 정의가 곧 데이터 및 평가 지표 결정)<br>
→ 모든 단계는 파이토치의 for epoch(반복횟수) in range(): 루프 안에서 순차적으로 실행<br>
입력(X): 데이터 주입 → 예측(Yp): Model forward → 손실(Loss): 오차 계산 → 역전파: 기울기(Grad) 계산 → 수정(업데이트): Optimizer Step
이후 입력으로 복귀

### 경사하강법: 
1. 현위치 탐색: 동서남북 1미터씩 이동 각각 고도변화 측정 
2. 최적 방향 결정: 가장 가파른곳 설정 
3. 이동 및 반복: 이후 일정거리 이동 후 다시 동서남북 이동시도<br>
현재 고도: Loss, 발의 기울기: 그래디언트, 보폭: 학습률(너무 크면 지나치고, 너무 작으면 너무 느림)<br>
최적의 파라미터를 계산하는게 아니라 찾아가는 것 → 반복 학습

$$예측 → 손실 계산 → 경사 계산 → 파라미터 업데이트$$

역전파: 미분 단계별로 쪼개 곱셈으로 연결하는데 이는 손실 발생에 대해 각 가중치가 얼마나 기여했는지 계산하기 위함.<br>
결과에서 시작해 원인 방향으로 정보 전달<br>
$$loss = criterion(y_pred, y_true) → 예측과 정답 비교, 차이가 손실$$
파이토치 역전파 → loss.backward() → optimizer.step()

활성화 함수 없으면 선형함수 + 선형함수 = 선형함수, 결국 층을 쌓아도 깊은 신경망 효과가 없음,<br> 
그래서 표현력 부족으로 복잡한 학습 불가 이미지, 자연어같은 비선형 데이터 처리가 불가능 → "Deep" 러닝이 안됨<br>
결론: 활성화 함수의 역할 → 비선형성 부여, 학습 안정성과 수렴에 영향<br>
대표 활성화 함수: 시그노이드(기울기 소실 현상 발생); Tanh(Hyperbolic 탄젠트, 여전히 기울기 소실현상 발생),<br>
ReLU(Rectified Linear Unit, 희소성 확보 및 연산 효율성 우수, 양수 영역에서 기울기 소실 완화, 그러나 음수 입력시 뉴런 사망(0이기 때문))

신경망 구조: Linear(선형 Wx+b) → Activation(비선형 함수 ReLU, Tanh) → Linear(비선형에서 특징 추출) → Activation(결정 경계 형성) → 반복<br>
Linear → Activation 반복 구조가 다층 퍼셉트론, CNN, Transfomer 모두에 공통 사용

### MSE 함수 정의
$$loss = ((Yp - Y)**2).mean() → loss.backward() =→ optimizer = optim.SGD([W, B], lr=lr)$$

### 국소 최적해
grad가 0인 지점이 2개일때 더 높은곳에 갖히는 경우(예방: Momentum 부여해 스스로 탈출할 수 있게 함)

키가 크면 남자라는 경향성이 있다고 할 때:<br>
단순 모델 → 편향이 높음: 편향(패턴을 제대로 학습 못함. 예: 키 크면 무조건 남자?)<br>
복잡한 모델 → 분산이 높음: 분산(노이즈까지 학습해버림. 예: 170.3은 남자 근데 170.2는 여자?)<br>
과소 적합(Underfitting): 데이터 패턴을 학습 못함 (Train 오차 높음 / Val 오차 높음) 편향 높음<br>
과적합: 노이즈까지 암기 (Train 오차 낮음 / Val 오차 높음) 분산 높음

### 데이터 분할
분할 이유: 시험 준비 (훈련 세트(교과서), 검증 세트(모의고사), 테스트 세트(진짜 수능))<br>
**핵심** 테스트 세트, 검증 세트 누수되면 망함: 검증과 평가 데이터 절대 훈련할때 직접 주입되거나 조작되어서는 안됨<br>
대용량 데이터(10만개 이상, 80:10:10 훈련:검증:테스트), 중소규모(1만개 이하, 60:20:20 훈련:검증:테스트),소량(천개 미만, K-Fold 교차검증)<br>
K-Fold 교차검증: 데이터 K개로 나누고 돌아가면서 검증, 적은 데이터로도 충분히 활용 가능
예: 1000개 데이터 있을때, 200개씩 분할해서 5 그룹으로 쪼개고, 첫번째는 1그룹 테스트 2345그룹 학습, 두번째는 2그룹 테스트 1345 학습,
이러면 5번의 별개 학습을 돌아서 총 4천번의 학습과 1000번의 시험을 본 인공지능 탄생, 5개의 평균이 최종 성능

### 혼동행렬
|||예|측|
|:---|:---|:---|:---|
|||A|B|
|실|A|TP|FN|
|제|B|FP|TN|

|모델|평가|
|:---|:---|
|회귀|MAE(비즈니스 보고용 최적)|
|균형분류|Accuracy  → (TP+TN)/전체|
|불균형분류|F1-Score() → 2*(Precision * Recall) / (Precision + Recall) → 정밀과 재현의 조화평균|
|의료진단|Recall    → TP / (TP+FN)|
|스팸필터|Precision → TP / (TP+FP)|

핵심 용어: 레이어 함수(모델 만드는 부품), 파라미터(학습통해 값 조정되는 데이터, 가중치 편향), 머신러닝 모델(합성 함수), 학습(파라미터 값 조정하는 행위)<br>
inplace=True → 자동 업데이트<br>
3대 핵심 함수이자 3대 요소: 예측함수(net, 학습 데이터 받아 예측값 출력), 손실함수(criterion, 예측값과 실제값 비교해 오차 계산), 최적화 함수(optimizer, 계산된 손실 바탕으로 모델 파라미터 정답에 더 가까워지는 방향으로 수정)<br>
$$학습 4단계 순환: 예측 계산 → 손실 계산 → 경사 계산 → 파라미터 수정 → 예측 계산$$
SGD(Stochastic Gradient Descent): 전체 데이터를 사용하는 대신, 무작위로 골라낸 일부 데이터만 가지고 gradient를 계산하고 모델을 업데이트

### 학습률 스케줄러
새 가중치 = 기존 가중치 - 학습률 * 그래디언트 → w1=w0-lr(δL/δw) → 초반에는 큰 학습률로 빠르게 진행, 후반에는 작은 학습률로 정밀하게 조정<br>
StepLR - 계단식 감소, 장점: 간단하고 직관적, 단접: 급격한 변화로 불안정할 수 있음<br>
CosineAnnealingLR - 부드러운 곡선, 장점: 안정적이고 범용적, 적용: 이미지 분류 등에 효과적<br>
OneCycleLR - 최신 기법, 장점: 가장 효과적인 최신 기법, 적용: 대규모 학습에 최적<br>

### 가중치 초기화
가중치 초기화 안하면 학습 자체가 불가능해짐(0점 조절이 안되어있는 저울)<br>
모든 가중치 0으로 → 모든 뉴런이 똑같이 학습되는 대칭성 문제 발생(뉴런이 3개 있어도 1개처럼 작동)<br>
가중치 너무 크게 → 활성화 값 폭발하고 활성화 함수 포화상태 도달(그래디언트 소실로 학습 멈춤)<br>
가중치 너무 작게 → 층을 통과할때마다 제곱을 하는데, 너무 작아져서 0이되면 신호가 완전히 사라져 정보 전달이 안됨

### 활성화 함수
비선형성 부여, 층 쌓기, 현실 데이터 학습<br>
Xavier vs He(Kaiming)<br>
He 써라<br>
가중치 = He(Kaiming)

배치 정규화: 내부 공변량 변화 해결, 강력한 학습 안정성<br>
풀 배치 vs 배치 정규화<br>
풀 배치하면 정확하게 그래디언트 계산하고 부드럽게 수렴하는데, 메모리 터지고 느림, 국소 최소값에 갖힐 위험도 있음<br>
배치 정규화하면 내부 공변량 변화를 해결할 수 있고, 강력한 학습 안정성이 있으나, 배치 잘못 설정시 학습도 불안정하고 메모리도 절약 못할수 있음<br>
그래서 배치 사이즈 어떻게 정하냐? 2**n으로 줄이거나 늘리면서 경험적으로 따져봐야함 명확한 답은 없음ㅋㅋ(존나 무책임함)<br>
레이어 정규화 != 배치 정규화



# 06장: 손실함수(Loss) 설계
손실함수: 모델의 예측값과 실제 정답 사이의 차이 수치화 하는 도구(시험 채점, 학습의 나침반)
손실함수 값이 감소하는 방향으로 가중치를 조정하면서 모델이 점점 더 정확한 예측을 하도록 학습함

### 회귀용 손실 함수
MSE(평균제곱오차) =(1/n) x Σ(y_true - y_pred)**2 → Σ의 의미: 모든 점.<br>
장점: 수학적으로 다루기 쉬움, 미분 가능, 빠른 학습 속도<br>
단점: 이상치에 매우 민감, 큰 오차 하나에 전체 손실 증가<br>
MAE(평균절대오차) = (1/n) x Σ|y_true - y_pred|<br>
장점: 강건성 → 모든 오차를 동등하게 취급해 이상치에 덜 민감<br>
단점: 0 근처에서 미분 불가

### MSE와 MAE의 장점 결합 = Huber Loss<br>
작은 오차는 MSE, 큰 오차는 MAE 처럼 동작함. δ(델타: 경계값 하이퍼 파라미터)값을 기준으로 두 방식 전환<br>
작은 오차에서는 제곱함수 사용해 빠르게 수렴, 큰 오차에서는 선형함수 사용해 이성치에 강건<br>
더 민감한 설정(0.5), 표준 설정(1.0) 권장, 더 강건한 설정(2.0)

### 회귀 문제 손실 함수 비교
|손실함수|이상치 민감도|수렴 속도|해석성|추천 상황|
|:---|:---|:---|:---|:---|
|MSE|매우 높음|빠름|중간|일반 회귀, 이상치 없음|          
|MAE|낮음|느림|높음|이상치 많음, 해석 중요|
|Huber|중간|중간|중간|균형필요, '로봇공학'|
|MAPE*|높음|중간|높음|비율 오차 중요|
|LogCosh|중간|빠름|낮음|MSE 대안|
MAPE(Percentage)

### 이진 분류☆: Binary Cross Entropy(BCE) 원리 이해
이진 분류: 두개의 클래스를 구분하는 손실함수(0또는 1 두가지 클래스를 분류할 때 사용)<br>
$$BCE = -[y * log(p) + (1-y) * log(1-p)] ; y: 실제 레이블(0 또는 1), p: 예측 확률(0~1 사이)$$
y=0이면 앞쪽 날아감, y=1이면 뒤쪽 날아감(판별식)<br>
확신도가 핵심 → 정답을 확신할수록 낮은 손실, 틀린 답에 확신하면 높은 손실 부여<br>
(p가 0 or 1이면 log값 미쳐 날뛰는 수치 안정성 때문에 실무에서 작은 엡실론(1e-7)값을 더해줘 안정성 확보함)

BCEWithLogitsLoss 원리 이해 → Sigmoid와 BCE의 수치적 불안정성 해결<br>
문제 상황 → Sigmoid 활성화 후 BCE 적용시 극단적인 값에 수치 불안정 발생.<br>
해결책 → BCEWithLogitsLoss는 시그모이드와 BCE 합치고 log-sum-exp 트릭으로 안정성 보장<br>
추가 장점 → 더 빠르고 효율적인 계산, 출력층에 시그모이드 넣지말고 그냥 raw logits 사용<br>
※출력층에 Sigmoid 사용 말고 그냥 BCEWithLogitsLoss 사용

### 다중분류: Cross Entropy Loss 동작 방식
파이토치에서 Softmax 내장<br>
Raw Logits: 모델 출력 [2.0 1.0 0.1] → Softmax: 확률로 변환 [0.66 0.24 0.10]→ Log: 로그 계산 [-0.41, -1.43, -2.30]→ NLL: 정답 인덱스 선택 최종손실 계산 → Raw Logits<br>
※입력 형식 주의: 원-핫 인코딩X 클래스 인덱스만 전달

모델 과신 완화: Label Smoothing 기법 이해<br>
모델 과신 문제: 일반적인 학습시 모델이 정답에 0.99이상 확률 부여하며 지나치게 확신, 이는 일반화 능력 저하시키고 새로운 데이터에 취약하게 만듬<br>
예시: 시험 공부(이 문제는 100% 정답이 1번이라고 외우기만 하면 융통성 없는 학습, 조금만 바뀌어도 무조건 틀림)<br>
라벨 스무딩: 정답 레이블 부드럽게 만들어 모델이 적당한 확신 갖도록 한다. [1,0,0] → [0.933,0.033,0.033]<br>
y_smooth = y*(1-α) + α / K ; α: 스무딩 파라미터(보통 0.1), K: 클래스 개수<br>
일반화 성능 향상 = 과적합 방지, 예측 확률 신뢰도 증가

### 클래스 불균형 문제
클래스 불균형 해결 전략 비교: Weighted Cross Entropy와 Focal Loss☆ 이해<br>
의료 진단: 정상 99%, 질병 1%일때 모델은 모두 정상이라고 예측해도 정확도 99퍼임, 그런데 쓸모 없지<br>
가중 교차 엔트로피(Weighted Cross Entropy): 소수 클래스의 손실에 더 큰 가중치 부여<br>
w_c = 전체 샘플 수 / (클래스 수 * 클래스 c 샘플 수) ; 예시: 클래스 0(9900개) 가중치 0.505, 클래스 1(100개) 가중치 50.0<br>
Focal Loss☆: 쉬운 샘플 손실 줄이고 어려운 샘플에 집중<br>
Focal Loss = -α * ((1-p)**γ) * log(p) ; 정답 확률 0.9면 손실 거의 무시, 0.1이면 손실 크게 유지<br>
Focal Loss는 극심한 불균형(1:100 이상)에 효율적, 객체 탐지나 의미론적 분할에서 자주 사용<br>
γ=2, α=0.25가 표준 설정

산악 지형처럼 높낮이 존재, 목표는 가장 낮은 계곡 찾기<br>

# 07장: 선형 회귀
문제 정의
회귀: 출력값이 '연속적인' 수치 → 입력값에 대응하는 정확한 수치에 근사 하는게 목표. 얼마나 많이(How much)
분류: 출력값이 정해진 카테고리(Label)중 하나 → 데이터 구분하는 결정 경계(Decision Boundary) 찾기가 핵심. 무엇인가?(Which one)

선형함수: 딥 러닝의 기본 블록,파이프 라인.
수학적 정의: y = Wx + b; 레시피 조절기: 커피 마실때 샷추가(weight), 설탕(혹은 우유)=편향; 핵심 인자
학습 파라미터: W(Weight),b(Bias)
차원(Dimensions): 입력이 벡터면 가중치도 벡터
모델의 가정: 입력 x 와 출력 y 사이의 선형 관계를 가정

최소제곱법(OLS)
목적: 오차의 제곱합(Sum of Squared Errors)를 최소화
제곱의 효과: 오차를 제곱하기 때문에 큰 오차(이상치)에 대해서 훨씬 더 큰 벌점(Penalty) 부여

딥러닝 관점 선형회귀
선형 회귀 = 뉴런 1개짜리 모델
연산 그래프: 모델 연산= z = Wx + b 손실 계산 L=MSE(y, z)
학습 원리: 역전파; 학습 고리→ 입력→ 예측 → 손실 → 역전파 → 입력값 업데이트 → 입력
딥러닝과 머신러닝의 차이: 딥러닝은 머신러닝의 일종이나 머신러닝은 선형 데이터만 학습 가능하고 딥러닝은
비선형 데이터도 학습이 가능하다
모듈 구조: nn.Module 상속 받아 구현
학습 파라미터: .weight(가중치), .bias: 편향, (requires_grad=True)
입출력 차원: 데이터 특징 수와 일치하도록 in_features와 out_features 지정

커스텀 클래스 이용한 모델 정의
l = nn.Linear(2, 3) → Linear(in_features=2, out_features=3, bias=True)
requires_grad=True 있으면 학습 대상임을 나타냄

nn.Module: 모든 모델 시작점
__init__메서드: 모델 뼈대 구성, 신경망 레이어 '선언'하고 초기화, 건물 건설 재료 목록 리스트와 같다
forward 메서드: 데이터 모델 통과하는 '흐름' 정의;
입력 데이터 어떤 순서로 어떤 계산 거쳐 최정 예측값으로 변화되는지 구체적으로 코딩,
각 방 및 복도 어떻게 연결되어 있는지 그리는 설계도

MSELoss 클래스 이용한 손실함수
예측값과 실제값 차이 평균 제곱, '미분 가능'해 경사 하강법 적용 용이, 오차를 제곱하니까 큰 오차에 큰 벌점 부여
실제값 변수 단위에 따라 손실 크기가 달라지므로 데이터 표준화 권장.

데이터 준비: 보스턴 주택 가격
모델 정의
무엇을 찾고 싶은지 정의
경사하강법
예측 계산 → 손실 계산 → 경사 계산 → 파라미터 수정. 학습 루프에 넣는거임
결과확인
학습 곡선 시각화해 우하향 곡선 나오는지 확인
중회귀 모델로 확장
학습률의 변경
학습률 0.01로두면 손실값이 inf→nan으로 발산 → 학습률 0.001로 해결

편향(bias)와 더미 변수의 관계
편향을 별도로 두는 방식과 더미 변수 사용하는 방식은 표기법 차이, 근본적 같은 계산 수행
파이토치 방식: y = wx + b (가중치 w와 바이어스 b를 별개로 취급)
더미 변수 방식: y = (w₁, w₀) · (x, 1) (바이어스 b를 더미 변수 1에 곱해지는 가중치 w₀으로 취급)
파이토치가 바이어스를 별도로 처리하는 이유는 
사용자가 입력 데이터 x에 매번 1이라는 더미 변수 열을 추가해야 하는 번거로움을 덜어주기 위함.

R**2 Score: 정의와 수식
R**2: 설명력(전체 데이터 변동 중 모델에 예측으로 설명한 비율, 1.0에 가까울수록 완벽)
상대적 성능 기준: 단순 평균 모델(R**2=0)대비 개선도 측정, 음수 나오면 평균보다 못한거니 망함

과적합 문제: 주요 징후(Train Loss는 줄어드는데 Validation Loss가 점점 증가함)
해결 방안: 배치 정규화, 데이터 증강, 조기 종료(Early Stopping), 드롭아웃, L2 정규화

모델 정규화
Ridge Regression(L2)
가중치 제곱합 패널티로 부과해 과적합 억제
Loss = MSE+λ||W||**2 → 
"모든 특성을 조금씩 사용하자", 계수를 0으로 만들지 않음, 잡음(Noise) 민감도 감소
언제 사용?: 다중공선성 문제 해결, 안정적인 예측 필요 시, 특성 제거 없이 규제

### 이진 분류

### 정확도
회귀 모델 평가: 손실이 얼마나 작은지가 중요했지만, 그 값이 '얼마나' 작아야 좋은 모델인지 직관적으로 알기 어려움
분류 모델 평가: 예측 정답과 맞는지 틀렸는지만 판단하면 되니 성적 매기기 훨씬 쉬움

### 훈련 데이터와 검증 데이터
어떤 학생이 족보 문제만 달달 외웠다고 치면, 족집게 시험에서 100점 맞지만, 처음 보는 문제가 나오면 다 틀릴것
딥러닝 모델도 너무 학습 데이터만 익숙해지면 새로운 데이터에 잘 안맞게 되는데 이것이 '과학습(과적합)'.
훈련 데이터: 모델이 학습하는데 사용(연습 문제), 검증 데이터:성능 평가에만 사용(모의고사)

### 시그모이드: 모델
f(x)=1/(1+exp(자연로그)*(-x))

### 교차 엔트로피: 손실
확신도가 높을수록 틀렸을때 더 큰 패널티 부여

# 09장: 다중 분류
다중 분류
이진 분류가 코인토스였다면, 다중 분류는 다지선다 시험 풀기, 선택지 많아진 만큼 더 정교한 전략 필요
모델은 모든 선택지에 대한 확률값을 모두 제시해야함(선택지 3개라면 확률도 3개, 합치면 100%)

여러 개의 분류기
여러 개의 분류기가 한 모델 내부에 들어있고, 이 모든 분류기가 입력 데이터 하나에 대한 확률값을 계산
이 중 가장 높은 확률을 제시하는 분류기의 그룹을 최종 예측값으로 선택

가중치 행렬
이진 분류                                                   다중 분류
가중치 벡터 사용                                            가중치 행렬 사용
하나의 레시피                                               여러 레시피
입력 벡터와 가중치 벡터의 내적으로 하나의 출력 계산          입력 벡터와 가중치 행렬의 곱셈으로 여러개 출력 동시 계산
출력이 1개에서 N개로 늘어남에 따라, 모델의 파라미터(가중치) 형태도 바뀜
동일한 재료(입력)를 가지고 각기 다른 가중치 행을 적용해 여러 출력을 각각 계산

소프트맥스 함수
이진 분류에서 시그모이드 함수 썼다면 다중 분류는 소프트맥스 함수
원시 출력(Logits)를 확률(%)로 바꿔줌 → 이 확률의 합은 1(100%)

교차 엔트로피 함수
다중 분류에서 교차 엔트로피 쓰는데 계산방식에 차이 있음
로그 적용 → 정답 요소 추출 → 손실 계산

모델 정의
데이터 분할 → 과적화 방지(모든 데이터 주면 모델이 데이터 단순하게 외울수 있음)
데이터 시각화 → 산점도 통해 데이터 분포 확인
텐서 변환 → 넘파이 배열은 파이토치 텐서로 변환(GPU 쓰려고), 입력 데이터 float, 정답 레이블 반드시 long(정수) 타입 지정☆

torch.max로 예측값 해석
torch.max(outputs, 1) 사용법
outputs 텐서에 대해 두번째 차원(dim=1, 행별로)을 기준으로 최댓값 찾기
반환값: values 실제 최대값 담긴 텐서, indices 최대값이 위치한 인덱스가 담긴 텐서
최종 예측: 우리가 필요한건 indices → torch.max(outputs, 1)[1]

학습 곡선으로 성능 모니터링
손실 곡선: 이상적인 손실 곡선은 반복 횟수 늘어날수록 부드럽게 감소해 낮은 값으로 수렴
정확도 곡선: 손실과 반대모양, 부드럽게 증가 높은값 수렴
안정성 확인: 훈련 손실 vs 검증 손실해서 비슷하면 안정적 학습

모델 출력값 확인
원시 출력(Logits): 직접 출력하는 값, 확률이 아니라 읽기 어려움
[2.1 0.8 -1.3]
소프트 맥스: 0과 1 사이의 확률값으로 변환, 모든 값 합 1
[0.65 0.30 0.05]

NLLLoss 함수의 정체와 필요성
NLLLoss: Negative Log Likelihood Loss, 교차 엔트로피 손실 계산의 핵심 부분 담당
역할: 정답 요소 추출, 정답 클래스에 해당하는 예측값만을 선택해 손실 계산
모델의 Logits에 소프트맥스 적용해 확률 구한 후 로그 취함
NLLLoss는 이 로그 확률 중 실제 정답 레이블이 가리키는 위치 값만 뽑아 음수를 붙여 손실로 사용
예: 정답 레이블 [0,1,2]에 대해 모델이 [log_prob_0, log_prob_1, log_prob_2]
반환했을때 NLLLoss는 -(log_prob_0 + log_prob_1 + log_prob_2) / 3 계산
파이토치는 정답 레이블에대한 원-핫 인코딩 전처리 과정 필요없음

NLLLoss에서 확률값이 0~1사이라 로그를 취하면 음수가되는데 이를 강제로 양수로 만들고 정답 라벨이 가리키는 인덱스값이 손실

다른 구현 패턴
패턴 1: 선형 모델 + CrossEntropyLoss (권장)
패턴 2: LogSoftMax + NLLLoss (패턴 1과 동일)
패턴 3: Softmax + 수동 log + NLLLoss (수동 방식, 권장 안함)

예측 라벨을 얻는 방법 ☆☆☆

질량함수는 막대그래프 밀도함수는 히스토그램

Batch Normalization(배치 정규화): 각 층의 출력 정규화 해 학습 안정화, 더 높은 학습률 사용 가능, 초기 가중치에 덜 민감해짐, 정규화 효과 있음
Learning Rate Scheduler: 학습 중 학습률 동적으로 조정 (ReduceLROnPlateau: 검증 손실 개선 멈추면 학습률 감소)
Ealry Stopping: 검증 손실이 개선되지 않으면 학습 조기 종료(과적합 방지, 학습시간 절약, 최적 모델 자동 선택)
혼동 행렬 해석: 대각선 올바른 예측, 비대각선 잘못된 예측
Precision vs Recal: 예측이 얼마나 정확한가(스팸필터) vs 실제 양성을 얼마나 잘 찾아내는가?(환자 탐색)
F1-Score: Precision과 Recall의 균형, 불균형 데이터셋에서 유용, 1에 가까울수록 좋은 성능



# 11장: CNN 기반 이미지 분류
CNN(Convolutional Neural Network, 합성곱 신경망): 선, 모서리, 질감 같은 특징들을 조합해 인식하는 딥 러닝 모델(분류기)
핵심 구조: 특징 추출(이미지에서 중요 패턴, 특징 찾아내는 단계)→정보 압축(핵심 정보만 남기고 불필요한 세부사항 제거)
→최종 분류(추출된 특징을 바탕으로 이미지 무엇인지 판단)
CNN의 가장 큰 두 부분→특징 추출(Features)+분류기(Classifier)→비유: 레고 조립 전문가
합성곱(특징을 찾아내는 툴→커널(커널의 모양이 달라지면 스캔 결과도 달라짐), 특징 뽑기)→
풀링(대표적인 부품 하나만 모아 전체 부품수 줄이고, 남은 부품 뭐있는지 간략하게 요약, 크기 줄이기)→
전결합층(모인 핵심 부품 조합보고 무엇(대상)의 재료가 되는지 최종 결론 내림, 최종 판단)
합성곱 처리: nn.Conv2d→필터(커널)가 이미지 한 칸씩 이동하면서 이미지 특정 부분과 커널의 값들 곱하고 모두 더해 하나의 결과값 만듬
이 과정 통해 이미지 (모서리, 질감, 색상 패턴→스타일) 등과 같은 저수준(low-level) 특징 추출 가능
커널의 값→파라미터(가중치)         [1x1 커널 => 필터 자체를 줄여서 핵심 정보만 쏙쏙 뽑아내게 함]
7*7 커널 보다 3*3 커널 쓰는 이유 = 7*7은 한 번만 돌려도 연산량 49인데, 3*3은 3번 돌려도 연산량 27, 즉 추상화를 더 잘하면서 연산량도 줄어들기 때문
커널이 이미지 스캔→특징 맵(Feature Map)생성, 학습 가능한 파라미터, 공간정보 유지
값이 급격하게 변함: 엣지(피처 아님!)

풀링
크기 축소(특징 맵의 크기를 줄여 계산량 감소)→강인함 확보(강인함 확보)→최대 풀링(특정 구역(예: 2*2)에서 가장 큰 값 선택)
nn.MaxPool2d: 합성곱 거친 특징 맵 크기 줄여줌. 특정 구역에서 가장 큰 값만 남기고 나머지 버림. 이미지 내 특징의 위치 약간 바뀌더라도 대응 가능한 강인함 얻게됨.

1차원 텐서화(Flatten Layer): 다차원에서 1차원으로 바꿔줘야함. 합성곱과 풀링을 거친 데이터는 채널, 세로, 가로(C,H,W)정보를 가진 다차원 텐서
그런데 최종 분류 담당은 선형함수(nn.Linear)라 1차원 벡터 형태 입력 받음.
nn.Flatten이 이 다차원 함수를 1차원으로 펼쳐줌

CNN 핵심 레이어
conv1 = nn.Conv2d(3, 32, 3) # 입력 채널: 3(컬러사진), 출력 채널: 32(특징맵), 커널 크기: 3x3
relu = nn.ReLU(inplace=True) # 활성화 함수 ReLU
conv2 = nn.Conv2d(32, 32, 3)# 입력 채널: 32, 출력 채널: 32, 커널 크기: 3x3
maxpool = nn.MaxPool2d((2, 2)) # 풀링 영역 크기: 2x2

채널 유지, 크기 30→28 (커널 적용)              채널 유지, 크기 절반으로 축소
[N개의 데이터, RGB, 32(x=,)32]

CNN 모델 클래스 설계

nn.Sequential: 관련 레이어 하나로 묶어 features(특징 추출)하고 classifier(분류)로 역할 분담

Flatten 입력 차원 계산
features 모듈 통과한 데이터가 결국 classifier의 첫번째 Linear 레이어에 들어갈때 그 크기는 얼마인가?

# 12장: 튜닝/최적화 및 과적합 대응
CNN 모델 아키텍쳐 설계: 풀링 통해 이미지 크기 절반 될때마다 채널수 2배로 늘려 정보 손실 최소화 후 더 많은 특징 학습

- 최적화 함수: 딥러닝 모델 학습시키는 주체. 손실 함수가 알려주는 가장 낮은 손실값을 향해 경사 정보 보고 파라미터 조작해 나아가는 역할
- SGD: 가장 기본적인 최적화 함수, 직관적이지만 때로는 비효율적, 국소 최적값에 갇힐 위험이 있다.
- Momentum: 관성 도입해 SGD 개선, 과거 경사 기억해 현재 경사와 합쳐 사용, 국소 최적값 탈출 가능, 빠르게 수렴
- Adam: 가장 널리 쓰임, 모멘텀+다른 알고리즘들 장점, 자동 학습률 조절, 안정적 성능

### 과적합(overfitting) 이해: 훈련은 잘하는데 검증에서 망하는 현상
#### 과적합 대응 방법:
- 드롭아웃: 신경망 뉴런 일부 무작위 비활성화, 네트워크의 특정 뉴련 과도 의존 막고, 전체적으로 더 강건하고 일반화 성능 좋은 모델 만듬.
- 배치 정규화: 각 미니배치 단위로 데이터 평균 0, 분산 1 정규화 해 다음 층으로 전달, 데이터 분포 안정시켜 학습 효율 높이고 과학습 예방
- 데이터 증강: 가지고 있는 데이터 인위적 변형해 데이터 양 늘림. 이미지 뒤집거나 회전시키거나 밝기 조절해 모델이 매번 조금씩 다른 데이터 학습.

#### 드롭아웃 동작 원리: 훈련모드, 평가모드 분리
훈련 모드에서는 확률로 뉴런 비활성화, 일부 요소는 0, 나머지는 0된 비율만큼 추가로 곱해줌(50% 비활성화 했으면 2배)

평가 모드: 드롭아웃 적용X 입력 출력 동일, 예측 시에는 모든 뉴런 사용해야 일관된 결과 얻을 수 있음.

층을 깊게 쌓은 모델 구현 및 학습: 층이 너무 깊으면 학습 효과가 입력층까지 전달되는 데 시간이 오래 걸릴 수 있다.

드롭아웃 적용 전략: 풀링 레이어 직후(풀디☆), 분류기 선형 레이어 사이, 점진적 비율 증가

배치 정규화
Conv 연산(특징 맵 추출)→배치 정규화(평균 0, 분산 1 정규화)→ReLU 활성화(비선형성 추가); 콘배렐루☆
합성곱 연산 직후, 활성화 함수 직전에 배치 정규화 배치
배치 정규화의 함정: 배치 정규화는 하나하나 파라미터와 내부 상태를 가진 레이어이기 때문에, 
동일한 bn레이어를 여러 위치에서 재사용하면 학습 과정이 꼬임. BN 레이어는 모델 내 모든 위치에서 고유한 인스턴스 생성해 사용해야함
정규화 → 분산을 줄임

데이터 증강: 사진 한장만 보여주지 않듯, 다양한 모습 보여주기

RandomHorizontalFlip: 무작위 좌우 반전, RandomErasing: 이미지 특정 사각 지역 무작위로 지움

핵심 정리: 최적화 함수 중요(Adam이나 Momentum 써야 SGD보다 좋음), 과학습 방지 기법 조합(드롭 아웃, 배치정규화, 데이터 증강 함께 사용),학습 시간과 성능의 균형 챙기기(학습 시간과 학습 횟수(epoch)가 더 많이 필요하지만 최종 성능은 우월하니 인내심 가지고 학습시키기)

# 14장: 사전 학습 모델(2)
### 3가지 전략:
1. 완전 동결(Full Freeze): 백본 전체 고정 후 분류기만 학습; 장점: 빠른 학습, 과적합 방지에 효과적; 단점: 도메인 차이 클 때 성능 제한
2. 부분 해제(Partial Fine-tuning): 초반층 고정, 후반 k개 층만 학습; 장점: 균형잡힌 접근, 안정적; 단점: k 값 선택이 중요
3. 전체 해제(Full Fine-tuning): 모든 층 학습 가능 성태로 전환; 장점: 최대 적응력과 유연성; 단점: 과적합 위험, 느린 학습

Full Freeze
```py
def setup_full_freeze(model):
     for param in model.parameters():
          param.requires_grad = False
     # 분류기만 학습
     model.fc = nn.Linear(model.fc.in_features, num_classes)
     return model
```
Partial Fine-tuning
```py
def setup_partial_finetune(model, k=1(k값은 알아서 정해야한다))
     # 일단 전부 얼리고
     for param in model.parameters():
          param.requires_grad = False

     # k 값에 따라 학습해라(ResNet stages)
     if k >= 1:
          for param in model.layer4.parameters():
               param.requires_grad = True

     if k >= 2:
          for param in model.layer3.parameters():
               param.requires_grad = True

     # 새로운 분류기
     model.fc = nn.Linear(model.fc.in_features, num_classes)
     return model
```

자언어 처리: 인간 언어와 딥러닝 연결하는 학문 분야
텍스트 전처리 파이프라인: 토큰화(문장을 단어로 분리) → 소문자 변환(대소문자 통일) → 불용어 제거(의미없는 단어 제거) → 패딩(동일 길이로 조정) → 어휘 구축(단어, 인덱스 매핑)

단어 임베딩
원-핫 인코딩 vs 밀집 임베딩
임베딩을 하면: 의미적 유사성 비슷한 단어들끼리 가까운 벡터 가짐; 차원 축소 만 단어 어휘를 128차원으로 압축; 관계 포착 '왕-남자+여자=여왕' 같은 의미 연산 가능
nn.Embedding의 역할:
패딩의 역할:임베딩된 단어가 숫자로 인코딩되어 정수 리스트로 모델에 들어가는데, 모델이 문장끼리 행렬 연산을 수행하려면 차원이 일치해야 병렬처리 가능하기 때문에 이를 위해 패딩 적용. 
최대 길이를 가진 문장(리스트)을 기준으로 그 보다 짧은 문장(리스트)의 끝이나 처음에 최대 길이를 가진 문장 만큼의 크기가 되도록 0으로 된 의미없는 값을 추가하는 것으로 동작.

LSTM 구조 이해
Long Short-Term Memory: 순환 싱경망 일종, "장기 의존성 문제☆" 해결하기 위해 고안됨. 기본 RNN이 겪는 기울기 소실 문제 게이트 메커니즘 통해 극복
입력 게이트(input gate): 새로운 정보 셀 상태에 얼마나 반영할지 결정 → 
망각 게이트(forget gate): 기존 셀 상태에서 어떤 정보를 버릴지 선택 → 
출력 게이트(output gate): 셀 상태에서 얼마나 출력할지 조절

휴먼 피드백 인 더 루프: 인간이 과정에 개입해서 라벨링 함

Transfomer로 전환
RNN(순환 신경망)/LSTM의 한계: 순차적으로 처리되니까 병렬화 어렵고, 긴 텍스트에서 속도 느림. 거리 먼 단어 간 관계 포착하는데 한계 있음
그래서 Transfomer: Self-Attention 매커니즘은 문장의 모든 단어가 서로 직접 참조할 수 있게 한다.
Query, Key, Value 벡터 통해 각 단어의 중요도 계산.
Query: 질문; Key:단어; Value:뜻 → 원래 query랑 key value 다른데 Self-attention에서는 query key value 값이 같아야함
